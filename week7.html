Week 7: Soil Moisture Detection & Data Logging using NodeMCU ESP32

1. Introduction

This project detects soil moisture using a sensor module and logs data to Google Sheets via Firebase using NodeMCU ESP32 and Google Apps Script.

2. Components Required

  • NodeMCU 32 ESP-12
  • Soil Moisture Sensor Module
  • Oscilloscope
  • Jumper wires, breadboard, power supply
  • Computer with Arduino IDE
  • Google Sheets & Apps Script

3. Understanding the Moisture Sensor Module

The sensor includes:

  • Probes: inserted into the soil
  • Module board: converts signals into analog output

Pin Descriptions:

  • VCC → 3.3V or 5V
  • GND → Ground
  • A0 → Analog signal output
  • D0 → Digital HIGH/LOW output

4. Measuring Moisture with Oscilloscope

  • Wet Soil: ~3–5V (High signal)
  • Moderate Soil: ~1.5–2.5V
  • Dry Soil: ~0.5V or less

5. NodeMCU Integration

Connect sensor output pins to GPIOs of NodeMCU. Analog values are read via ADC, and logic levels through digital pins.

6. Firebase and Google Sheets Integration

Store data in Firebase, then pull it into a Google Sheet:

  1. Create Google Sheet with headers
  2. Use Apps Script to fetch and insert data
  3. Set triggers to automate every minute

7. Conclusion

The system works reliably for live soil condition monitoring. Data flows from sensor → NodeMCU → Firebase → Google Sheets.

Soil Moisture Interpretation Table

  • 10–100: Sensor disconnected
  • 100–300: Wet soil (80–100%)
  • 300–600: Moderate (50–70%)
  • 600–900: Drying soil (30–50%)
  • 900–1200: Very dry (0–20%)
